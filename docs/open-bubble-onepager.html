Open Bubble

Route any screen to code, context, or action.

A screenshot leaves the Android phone, the OpenAI SDK classifies the intent, and Open Bubble sends it into the right Codex-powered lane.

Codex as runtime, not just copilot.
High-level flow

One screenshot. Three outcomes.

We use the OpenAI SDK to inspect what is on screen and decide whether the user is asking for a coding task, a context-graph question, or a personal assistant action.

  • Code lane opens the right repo and returns a PR.
  • Context lane queries a DuckDB local graph and answers.
  • Action lane uses Codex app-server plus Gmail and Calendar.

Coding task

Open the correct repo

Codex works in the matching codebase, makes the change, and returns a PR.

  • Repo-aware coding lane
  • End-to-end modify and iterate
  • Best for bugs, UI tweaks, and implementation work
Context graph question

DuckDB local context graph

The question is routed into a local context graph we built and populated with Codex and ChatGPT-derived context.

  • DuckDB-backed graph with entities, facts, and chunks
  • Local provenance-aware graph inspector
  • Codex is used to seed and grow the graph from connected data
Personal assistant action

Plugins plus confirmation

Codex app-server operates connected tools like Gmail and Calendar, then returns a concise confirmation.

  • Personal assistant lane
  • Gmail and Calendar workflows
  • Action result comes back as confirmation
